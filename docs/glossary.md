# Glossary

- **Architecture family**: A recurring design pattern (monolith, mesh, governance-layered, hybrid).
- **Framework**: A named instantiation of an architecture family with specific assumptions (e.g., Surak).
- **Primitive**: A reusable component that can appear across many frameworks (e.g., shards).
- **Scenario**: A checklist-able narrative of failure/behavior under conditions (e.g., optimizer pathology).
